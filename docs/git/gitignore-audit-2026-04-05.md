## Notes on Intentional Patterns

The following patterns were examined and confirmed intentional; they are not findings:

- **`Cargo.lock` ignored**: The `.gitignore` ignores `Cargo.lock`. For Rust *libraries* this is the recommended practice (consumers pin via their own lockfile); for Rust *executables* it should be committed. Since the planned Rust components are "performance components" (libraries linked into the extension host), ignoring `Cargo.lock` is correct. If an executable binary component is added later, this should be revisited.
- **`.vscode/` with negations**: The `.gitignore` ignores `.vscode/` but negates `launch.json`, `tasks.json`, `extensions.json`, and `settings.json`. This is intentional for VS Code extension development — these four files contain team-shared debugging and task configuration.
- **`package-lock.json` tracked**: Correctly tracked. Lock files for application packages (as opposed to libraries) should be committed for reproducible installs.
- **`CLAUDE.md` tracked**: Intentional — this is the project-level AI assistant instruction file, not a secret.
- **`settings.local.json` ignored**: Intentional — covers local override settings (Claude Code and similar tools write these per-machine).
